# Шпаргалка по работе с ветками в git

Проверка текущей ветки и статус репозитория:

```shell
git status
```

Операцию **status** следует обязательно использовать для контроля действий на разных этапах.

Создайте новую ветку и переключитесь на неё:

```shell
git checkout -b <имя_новой_ветки>
```

Можно разбить задачу на два этапа - сначала создать ветку:

```shell
git branch <имя_новой_ветки>
```

Затем переключиться на неё:

```shell
git checkout <имя_новой_ветки>
```

Проверка успешности переключения:

```shell
git branch
```

Текущая ветка будет отмечена звёздочкой (*).

## Слияние веток

Для слияния необходимо перейти в branch, в который будет осуществляться слияние (обычно - main). Современный вариант перехода на основную ветку (main):

```shell
git switch main
```

Контрольное действие - получить обновления из remote-сервера:

```shell
git pull origin main
```

Cлияние указанной ветки с текущей:

```shell
git merge <имя_ветки_из_которой_нужно_взять_изменения>
```

Конфликты разрешаются (resolve) вручную. В случае успешность, изменения следует внести и на удалённый сервер:

```shell
git push origin main
```

Есть три разных метода слияния:

- **Merge** — по умолчанию. Сохраняет всю историю, безопасен для публичных веток
- **Rebase** — для личной работы или небольших команд. Даёт чистую историю. Команда `git rebase main <имя_вашего_бранча>`
- **Squash** — когда нужно упростить историю, объединив множество мелких коммитов в один логический блок. Команда: `merge --squash`

## Удаление ветки

Чтобы удалить локальную ветку, следует использовать команду:

```shell
git branch -d <имя_ветки>
```

Параметр `-d` означает "безопасное удаление" - Git не даст удалить ветку, если в ней есть коммиты, которых нет в текущей ветке.

Принудительно удаление:

```shell
git branch -D <имя_ветки>
```

Удаление remote-верки на сервере:

```shell
git push origin --delete <имя_ветки>
```

Синхронизация локального репозитория с remote:

```shell
git remote prune origin
```

GitHub предоставляет возможность удалить remote-ветку через свой графический интерфейс.
