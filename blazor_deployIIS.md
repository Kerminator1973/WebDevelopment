# Deploy Blazor-приложения на Microsoft IIS

Для полноценной сборки (с оптимизацией кода) Blazor-приложения, на машине разработчика должна быть установлена **рабочая  нагрузка** wasm-tools.

>Рабочая нагрузка (workload) - это  метапакеты, которые определяют набор инструментов, SDK и инструментов сборки, необходимых для конкретных сценариев разработки (например, MAUI, Blazor WASM, Aspire). Они устанавливаются в контексте .NET SDK и не зависят от Visual Studio. Обычно устанавливаются через командную строку, используя `dotnet workload install`

Рабочая нагрузка wasm-tools, в часности, содержит Emscripten - компиляторы для Wasm.

Установить рабочую нагрузку можно командами:

```shell
dotnet workload install wasm-tools
dotnet workload install wasm-tools-net8
```

Посмотреть список установленных workload можно командой:

```shell
dotnet workload list
```

Сборка приложения Blazor WebAssembly может осуществляться как в режиме Ahead-of-Time (компиляция кода в Wasm), так и без компиляции. Во втором случае, IL-сборки .NET-кода будут интерпретироваться "на лету", т.е. код будет работать медленнее. Однако загружаться он будет быстрее и скорость компиляции в режиме Ahead-of-Time на порядок больше, чем без него.

Длительное время компиляции, скорее всего, связано с тем, что осуществляется ресурсоёмкая оптимизация Wasm-кода.

## Развертывание приложения на IIS 8.5

С точки зрения IIS, приложение Blazor WebAssembly - это просто обычный статический web-сайт. Описание параметров настройки этого сайта осуществляется в "web.config".

Однако, приложение может потребовать установик отдельного компонента URL Rewrite. URL Rewrite - это расширение IIS, движок перезаписи на основе правил — определяет логику анализа входящих URL и их преобразования.

## Проблема с подключением клиента к тестовому серверу

При попытке подключения клиента к тестовому серверу может возникнуть проблема подключения, отображаемая в Developer Console (F12):

```console
:9310/ruf:1  Failed to load resource: net::ERR_CERT_AUTHORITY_INVALID
blazor.webassembly.js:1 Возникла ошибка подключения к серверу
```

Проблема связана с тем, что на тестовом сервере используется само-подписанный сетификат. Современные браузеры очень не любят подобные сертификаты. Чтобы решение заработало, то нужно попытаться зайти на тестовый сервер через браузер по его URL:

```
https://192.168.100.71:9310/ruf
```

Для дальнейшей работы браузер потребует от пользователя явным образом разрешить использовать данный сертификат и, если пользователь его подтвердит, то браузер запомнит выбор и разрешит Blazor-приложению (`http://192.168.100.71:9312`) доступ к серверу.
