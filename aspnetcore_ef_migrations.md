# Работа с миграциями посредством Entity Framework

Добавить новую миграцию можно командой:

```shell
dotnet ef migrations add <ИмяМиграции>
```

Внести изменения в структуре базы данных можно командой:

```shell
dotnet ef database update
```

## Удаление миграций в базе данных и в коде

Проверка списка миграция в кодовой базе проекта осуществляется командой:

```shell
dotnet ef migrations list
```

Миграции, которые были созданы в проекте, но не опубликованы в конкретной базе данных (см. "appsettings.json") помечаются в выводе, как отложенные (Pending).

Удалить последнюю миграцию можно командой:

```shell
dotnet ef migrations remove
```

Повторять команду можно сколько угодно раз и инструмент будет удалять по одной миграции за раз. Однако, удаляются только миграции в статусе "Pending". Если нужно удалить миграции и из базы данных, и из кода, то сначала следует удалить их из базы данных, указав имя последней по времени, правильной миграции. Например:

```shell
dotnet ef database update NoDuende
```

Корректность удаления миграции из базы данных можно проверить запросом:

```shell
SELECT * FROM public."__EFMigrationsHistory"
ORDER BY "MigrationId" ASC LIMIT 100
```

После того, как миграции в коде будут отображаться в команде `dotnet ef migrations list` в состоянии Pending, можно начать удалять их из кода.
