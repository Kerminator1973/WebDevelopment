# Что такое ORM

ORM (Object‑Relational Mapping, объектно‑реляционное отображение) — технология, которая служит "прослойкой" между реляционной базой данных (БД) и объектно‑ориентированным языком программирования. Она позволяет работать с данными в БД так, будто это обычные объекты в коде, — без прямого написания SQL‑запросов.

Одна из основных проблем "сырого" SQL - типобезопасность. Для промышленных программистов желаемым свойством системы выполнения запросов к СУБД является синтаксическая и семантическая (соответствие схеме базы данных) проверки.

## Преимущества ORM

- Ускорение разработки. Не нужно писать много кода для преобразования данных между форматами БД и языка программирования
- Абстракция от СУБД. Можно относительно легко сменить базу данных (например, с MySQL на PostgreSQL) — достаточно поменять настройки ORM
- Единый интерфейс. Работа с данными через привычные методы языка программирования, без переключения на SQL
- Снижение числа ошибок. ORM автоматически генерирует безопасные запросы, уменьшая риск SQL‑инъекций
- Поддержка сложных связей. Легко отображать отношения между таблицами ("один ко многим", "многие ко многим"и т. д.)

## Недостатки ORM

- Накладные расходы на производительность. Автоматически сгенерированные SQL‑запросы могут быть неоптимальными и работать медленнее, чем ручные
- "Проблема N+1 запросов". При неправильной настройке ORM может выполнять избыточное количество запросов к БД
- Сложность отладки. Трудно понять, какой именно SQL‑код генерируется и почему возникают проблемы с производительностью
- Кривая обучения. Освоение ORM‑фреймворка требует времени, особенно для новичков
- Избыточность для простых задач. Для небольших проектов или простых операций ORM может оказаться слишком сложным решением

## Альтернативные решения

Одно из альтернативных направлений - использование специализированных DSL для работы с СУБД.

DSL (Domain-Specific Language) — это предметно‑ориентированный язык программирования, созданный для решения задач в конкретной области. В отличие от языков общего назначения (GPL, General‑Purpose Language — например, Java, Python или C++), DSL фокусируется на узком наборе проблем и предлагает оптимизированный синтаксис и семантику.

Пример DSL - LINQ из C#. Причины:

**Предметная ориентация**. LINQ создан для решения задач обработки данных: фильтрации, сортировки, группировки, объединения коллекций. Его функционал заточен именно под запросы к данным.

**Специальный синтаксис**. В C# и VB.NET LINQ добавляет новый синтаксис, напоминающий SQL:

```csharp
var result = from item in collection
             where item.Property > 10
             select item;
```

**Абстракция от реализации**. LINQ предоставляет единый интерфейс для работы с разными источниками данных:

- LINQ to Objects — запросы к коллекциям в памяти
- LINQ to SQL — работа с реляционными БД
- LINQ to XML — обработка XML‑документов
- другие провайдеры

**Трансляция в целевые языки**. Запросы LINQ транслируются в:

- вызовы методов IEnumerable/IQueryable — для коллекций в памяти
- SQL‑запросы — при работе с БД
- XPath/XQuery — для XML

**Тип DSL**. Это внутренний (встроенный) DSL — он использует синтаксические конструкции базового языка (C#/VB.NET), но создаёт специализированный язык запросов внутри него.

### SQLC в Go

SQLC (sqlc) — это инструмент для экосистемы Go, который автоматически генерирует типобезопасный код на основе SQL‑запросов. Он сочетает контроль над чистым SQL с удобством кодогенерации — и служит альтернативой ORM‑подходам.


Вы пишете SQL‑запросы в отдельных файлах, а sqlc на их основе генерирует Go‑код: структуры данных и методы для выполнения запросов.

Процесс работы:

- Описываете схему базы данных (в файле schema.sql)
- Пишете SQL‑запросы с аннотациями (в файле queries.sql)
- Запускаете команду `sqlc generate`
- Получаете готовые Go‑файлы с кодом

Ключевые особенности:

- Типобезопасность. Ошибки в запросах или несовместимость типов выявляются на этапе компиляции
- Высокая производительность. Сгенерированный код не использует рефлексию, минимизирует аллокации памяти и работает напрямую с database/sql или pgx
- Полный контроль над SQL. Вы пишете запросы вручную — нет "чёрного ящика", как в ORM
- Валидация запросов. SQLC проверяет запросы на соответствие схеме БД: наличие таблиц, колонок, корректность типов
- Поддержка миграций. Интегрируется с популярными инструментами: Goose, Atlas, golang‑migrate
