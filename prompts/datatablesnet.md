# Цель решения

Веб-интерфейс для просмотра и управления списком с пагинацией, поиском и сортировкой.

Необходимо сгенерировать следующие файлы: 

- index.html, HTML-верстка
- index.css, каскадные таблицы
- index.js, JavaScript-код

Все данные, необходимые для работы таблицы должны быть получены через API посредством http-запроса при открытии документа, загружены в память браузера.

Ключевые пользовательские сценарии:

- просмотр данных
- поиск по частично введённому тексту из ячеек таблицы
- сортировка по щелчку на заголовке таблицы

## Требования к поведению веб-интерфейса

Пользовательский интерфейс страницы должен содержать кнопки "Добавить", "Изменить", "Удалить".

Кнопки "Изменить" и "Удалить" должны быть доступны только в том случае, если в таблице есть один выбранный элемент.

При нажатии на кнопку "Добавить" должен отображаться модальный диалог с полями ввода, соответствующими колонкам таблицы. Значения в полях модального диалога должны быть пустыми.

При нажатии на кнопку "Изменить" должен отображаться модальный диалог с полями ввода, соответствующими колонкам таблицы. Значения в каждом поле должны соответствовать значениям взятым из текущей выбранной строки таблицы.

При нажатии на кнопку "Удалить" должен отображаться модальный диалог с кнопками "Подтвердить" и "Отменить".

При подтверждении действия в модальных диалогах должен быть выполнен запрос к API, выполняющий соответствующее кнопке действие.

## Технические требования

- Библиотека для отображения библиотеки DataTables.NET 2.3.x
- Для выбора элемента таблицы следует использовать plugin select из состава официальных зависимостей библиотеки DataTables.NET
- Базовая библиотека jQuery 3.7.x
- Библиотека стилистического оформления: Bootstrap 5.1.x
- Все библиотеки должны быть загружены в проект. Использование CDN не допускается, т.к. web-приложение будет работать в корпоративной сети без доступа в интернет
- Формат получения данных от API: JSON через AJAX-запрос
- Сортировка и поиск должны осуществляться без необходимости взаимодействия с API
- Браузерная совместимость: Google Chrome, Microsoft EDGE, Yandex.Browser
- JavaScript-код и каскадные таблицы (CSS) должны быть размещены в отдельных файлах

## Структура HTML‑документа

- Базовая разметка (`<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`)
- Подключение CSS и JS (DataTables, jQuery, Bootstrap)
- Секция `<script>` для инициализации DataTables (пустая заглушка с комментарием "сюда вставляется JS‑код")

## Спецификация таблицы DataTables

Таблица должна содержать поле с идентификаторов записи в базе данных. Поиск по этому полю должен быть запрещён:

```js
datatable_settings["columns"] = [
    {
        "data": "id",
        className: "dt-body-center",
        "searchable": false // Не ищем по номеру идентификатора
    },
];
```

Сортировка по умолчанию должна осуществляться по первому полю (идентификатор записи).

По умолчанию, на странице таблицы должно отображаться не 14 строк.
