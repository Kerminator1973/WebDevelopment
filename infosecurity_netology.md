# Информационная безопасность

Intrusion Prevention System (IPS) - система предотвращения вторжений, защитное решение, предназначенное для обнаружения и пресечения потенциально опасной активности в компьютерной сети или на отдельном устройстве.

Межсетевой экран (Firewall) - обеспечивает защиту периметра, фильтрует трафик по заданным правилам. Анализирует заголовки сетевых пакетов и сравнивает их параметры с набором правил (ruleset). Проверяемые параметры:

- IP‑адрес источника и назначения
- порт источника и назначения
- используемый протокол (TCP, UDP, ICMP и др.)
- флаги протокола
- состояние соединения

Осуществляет **контроль сессий** (stateful inspection):

- фиксируют установление сеанса связи в специальной таблице
- пропускают только пакеты, относящиеся к уже установленным соединениям
- отбрасывают пакеты, имитирующие пакеты завершённых сессий

Посредством **глубокого анализа пакетов** (Deep Packet Inspection, DPI) выявляет:

- вредоносный код
- подозрительные шаблоны данных
- попытки эксплуатации уязвимостей

>Проверка дистрибутива, или исполняемого файла несколькими анти-вирусами online: [VirusTotal](https://www.virustotal.com/gui/home/upload)

Политики безопасности определяют:

- какой трафик разрешён
- какой трафик запрещён
- какие действия выполнять с подозрительными пакетами

Ключевым для ИБ является система аутентификации.

Центр мониторинга безопасности (SOC = Security Operation Center) - управляющая консоль для контроля ИБ.

## Организация удалённой работы

Remote Access VPN - подключение к корпоративной сети. У удалённого сотрудника должны быть Антивирус и VPN-доступ. Одна из ключевых атак злоумышленников - DNS redirect. Рекомендуется устанавливать: Anti-Malware, DNS Security, Personal Firewall, Posture Assessment.

>Kaspersky Secure Connection - это не VPN для доступа к сервисам внешнего интернета, а защита "DNS Security" + VPN.

## Типовые системы ИБ

- Packet Filter: Анализирует пакеты до L4 включительно (IP-адреса и TCP/UDP-порты). Пример: iptables
- Firewall (stateful): Анализирует пакеты до L7. Может динамически открывать обратный трафик для сложных протоколов (FTP, SIP, и т.д.). FTP-сервер получает запрос от клиента, сообщает динамический адрес дальнешей работы и переключается на работу по этому порту - Packer Filter не справился бы с такой задачей
- Next Generation Firewall (NGFW): все функции Stateful Firewall, система предотвращения вторжений (IPS), сетевой антивирус с контроль файлов, расширенные возможности web-фильтрации (например, можно блокировать функцию Like на Facebook)
- Антивирусная защита. Бывает двух типов: поиск по сигнатуре, поиск аномалий

>Поиск аномалий:
>
>- Каждый исполняемый файл запускается в "песочнике" для анализа его активности (редактирование реестра, сетевые соединения и т.д.)
>- Инновационный подход. Системы называются песочницам (sandbox)
>- Может работать на уровне сети, или на конкретной рабочей станции
>- Существуют облачные сервисы

- Система обнаружения вторжений (только определяет и сигнализирует) и система предотвращения вторжений (может блокировать атаку). Могут работать как на рабочих станциях, так и на уровне сети
- Система защита web-трафика. Анализируются исходящий и входящий трафик. Для защиты исходящего трафика применяются **прокси-серверы** (работа с HTTPS, URL-фильтрация, фильтрация по типу файлов). Часто прокси-сервера устанавливаются по схеме "man in the middle". Для защиты входящего web-трафика применяются Web Application Firewall (WAF). Умеют блокировать _SQL Injection_ и _cross-site scripting_
- Системы защиты от DDoS-атак. Бывают "Волюметрические атаки" - переполнение канала, а также встречаются атаки на приложений с целью исчерпания ресурсов на сервере. Волюметрические атаки отражаются операторами связи, а атаки на приложения можно отражать на уровне корпоративной сети (устанавливаются перед web-сервером)
- SIEM (Security infromation and event management) и SOC (Security operation center)

## Типовая архитектура безопасности

Демилитаризованная зона (DMZ) - отдельный сетевой сегмент, который позволяет безопасно разместить сервисы, публикуемые в интернете. DMZ - это буфер между интернет и корпоратиыной сетью.

Особенно безопасной считается схема, в которой между интернетом и DMZ установлен firewall одного вендора, в между DMZ и внутренней сетью предприятия - firewall другого вендора.

Критически важные системы находятся в отдельном защищённом корпоративном сегменте, в котором используются модули доверенной загрузки ОС, усиленная криптография, IDS, и т.д. Доступ рядовых сотрудников в этот сегмент не осуществляется.

В финансовых системах могут использоваться anti-fraud-систему.

## Тестовое задание

Мои ответы на тестовое задание по курсу инфобеза:

Ритейл:

- #1 Реверсивный прокси, Firewall: разрешить использовать только https, а http для перехода на https
- #2 Систему защиты от DDoS-атак: следует разместить в корпоративной сети, но до web-сервера
- #3 Web Application Firewall, которые умеют анализировать трафик и блокировать SQL Injection и cross-site scripting

Промышленность:

- #1 Firewall: блокировать любые входящие соединения за исключением https, по фиксированным IP-адресам. Возможно имеет смысл заблокировать вообще все входящие соединения. Возможно, имеет смысл поставить два разных Firewall-а на обоих границах DMZ
- #2 Spam-фильтра: возможно, с ИИ, который будет динамически модифицировать правила
- #3 Антивирус: как на сервере электронной почты, так и на каждом рабочем месте сотрудника
- #4 Корпоративный прокси-сервер, который будет работать по разным схемам: полный доступ, белый список, минимальный доступ. Правила устанавливаются для каждого сотрудника
- #5 Firewall: рекомендуется отличный от Firewall-а на входе в DMZ
- #6 Антивирус, Firewall

#3 - для определения и блокирования сложных WEB атак на ваш интернет-магазин, когда злоумышленники используют SQL инъекцию для получения доступа к вашей базе клиентов. WAF защищает веб-приложения от атак на прикладном уровне (L7). WebTitan можно использовать на удаленных устройствах.

Промышленность

#1 для разграничения доступа из Интернета в DMZ

Межсетевой экран основные средства для контроля трафика между Интернетом, DMZ и внутренней сетью. Касперский NGFW

Антивирусные шлюзы и спам-фильтры системы сканируют входящую и исходящую электронную почту на наличие вредоносных программ. Kaspersky

#4 для контроля WEB трафика, URL фильтрации

Промышленный межсетевой экран : Специализированные решения для защиты технологических сетей и систем SCADA. Прокси серверы. ARMA подходит для производства

#5 для разграничения доступа между корпоративной сетью и DMZ

VPN-шлюзы для безопасного соединения между двумя сетями. Межсетевой экран основные средства для контроля трафика между Интернетом, DMZ и внутренней сетью.
